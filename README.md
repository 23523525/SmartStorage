## 1.3. Тема проекта

Тема: Информационная система управления складскими запасами «SmartStorage» (REST-сервис с использованием PostgreSQL).

## 1.4. Цель проекта

Цель проекта — разработать веб-службу для учёта складских запасов с поддержкой категорий товаров, остатков, истории движений и работы с QR-кодами, обеспечивающую удобный программный интерфейс (REST API) для интеграции с другими системами и клиентскими приложениями.

## 1.5. Задачи проекта

Для достижения цели решаются следующие задачи:

1. Спроектировать структуру базы данных PostgreSQL (таблицы категорий, товаров и движений, связи между ними).
2. Разработать REST-API на основе Spring Boot для управления категориями, товарами и операциями прихода/расхода.
3. Реализовать учёт остатков товара на складе с автоматическим пересчётом количества при каждом движении.
4. Добавить сервис работы с QR-кодами для идентификации товара и интеграции с внешними клиентами.
5. Обеспечить централизованную обработку ошибок и валидацию входящих данных.
6. Настроить миграции базы данных с помощью Flyway и документирование API через OpenAPI/Swagger.

## 1.6. Аннотация

Проект «SmartStorage» представляет собой серверное приложение на базе Spring Boot для управления складскими запасами. Система хранит информацию о категориях и товарах, фиксирует операции прихода и расхода в таблице движений, рассчитывает текущие остатки и предоставляет REST-интерфейс для внешних приложений. Данные сохраняются в СУБД PostgreSQL, структура БД разворачивается с помощью миграций Flyway. Дополнительно реализована работа с QR-кодами для быстрого доступа к информации о товаре. Документация API доступна через OpenAPI/Swagger.
